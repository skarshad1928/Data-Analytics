# 📊 End-to-End Financial Transaction Intelligence System

## 👨‍🎓 Role

**Student Project**
This project demonstrates how a single individual can perform the roles of:

* Data Engineer
* Data Analyst
* Data Scientist

---

## 🎯 Objective

To analyze financial transaction data, extract meaningful insights, and build a predictive model to detect fraudulent activities while simulating real-world data workflows.

---

## 🧩 Project Overview

This project is designed as a complete data lifecycle pipeline:

1. **Data Engineering** → Data extraction, cleaning, and transformation
2. **Data Analysis** → Visualization and insight generation
3. **Data Science** → Predictive modeling and decision support

---

## 📁 Dataset

* Financial transaction dataset (PaySim)
* Contains millions of simulated mobile money transactions

### Key Features:

* Transaction type
* Transaction amount
* Account balances before & after transaction
* Sender and receiver IDs
* Fraud labels

---

## ⚙️ Project Workflow

### 🟨 1. Data Engineering (10%)

**Goal:** Prepare clean and structured data

**Tasks Performed:**

* Data loading and inspection
* Handling missing values
* Removing duplicates
* Data type corrections
* Feature engineering:

  * Balance difference
  * Transaction categories

**Output:**

* Clean, reliable dataset ready for analysis

---

### 🟦 2. Data Analysis (70%)

**Goal:** Understand and interpret data

**Tasks Performed:**

* Exploratory Data Analysis (EDA)
* KPI generation:

  * Total transactions
  * Fraud rate
  * Average transaction value
* Pattern identification:

  * Fraud by transaction type
  * Time-based fraud trends
* Dashboard creation (Power BI)

**Visualizations:**

* Transaction trends over time
* Fraud distribution
* Transaction type comparison
* KPI summary cards

**Key Insights:**

* Fraud is more frequent in specific transaction types
* High-value transactions have higher fraud probability
* Fraud patterns vary over time

---

### 🟥 3. Data Science (20%)

**Goal:** Predict fraudulent transactions

**Tasks Performed:**

* Feature selection
* Model building:

  * Logistic Regression
  * Random Forest
* Model evaluation:

  * Accuracy
  * Precision
  * Recall

**Output:**

* Fraud prediction model
* Risk identification system

---

## 🔄 End-to-End Pipeline

Data → Cleaning → Analysis → Visualization → Prediction → Insights

This project simulates how data flows across different roles in a real-world organization.
